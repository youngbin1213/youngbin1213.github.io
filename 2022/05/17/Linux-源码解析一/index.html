Linux 源码解析（一）

Linux 源码解析
Linux 源码解析
Publish Date:   2022-05-17

目录

arch目录

针对处理器CPU的子目录

  • 在arch→arm→boot目录中，默认编译生成的内核镜像是在这个目录中。
  • 在arch→arm→kernel目录中，有针对具体CPU处理器的代码，有相关内核特性实现方式，如信号处理等。这一部分当然是芯片厂商做好了，4412的这部分就是三星已经做好的部分
  • arch -> arm -> lib 目录中，硬件相关的库函数，后面学习驱动时使用
  • arch -> arm -> tools 目录种，包含生成镜像的工具

内核开发的特点

  • 不能访问C库和标准的C头文件
  • 必须使用GNU C
  • 缺乏用户空间那样的内存保护机制
  • 难以执行浮点运算
  • 每个进程只有一个很小的定长堆栈
  • 支持异步中断、抢占和SMP，因此必须时刻注意同步和并发
  • 可移植性

无lib库，无标准头文件

C库太大，而且太低效了

举例 printk 和 printf 。printk是内核自己实现的，负责把格式化好的字符串拷贝到内核的日志缓冲区，这样，syslog程序就会读取缓冲区来获得内核消息。区别是printk能指定优先级

GNU C

内联函数

内联函数的优点不用多说了，（减少了函数调用的占空间等）

内联函数的缺点： 函数直接编译，占用更多的内存空间或者指令缓存，如果没有特别的时间上限，不建议写成内联函数

需要使用 static 关键字，和 inline 关键字限定

static inline void wolf(unsigned long tail_size)

内联汇编

GCC支持 在C函数中嵌入汇编指令。只有知道对应的体系结构，才能使用这个功能（这句话没懂

Linux在偏近体系结构底层或者对时间要求很低的，会使用汇编语言

分支声明

对分支选择进行优化

内核把 指令封装成了宏，比如 likely 和 unlikely

用法：

if (likely(success)){
    /* ... */
}

同步和并发

内核许多特性要求能并发地访问共享数据

  • Linux是抢占多任务多操作系统。内核的进程调度程序即兴对进程进行调度
